# breadmin-composer v0.1.x — Cold Start

**Theme:** The engine turns over for the first time.

## Overview

First working release of breadmin-composer. Implemented the full six-stage pipeline
(spec → plan-milestones → research-worker → design-worker → plan-issues → impl-worker)
as a single Python CLI package. Established the orchestrator/sub-agent architecture,
subprocess runner, checkpoint persistence, and the `composer` admin command.

## Goals

- All six pipeline stages operational as `composer` subcommands
- End-to-end pipeline execution against a target repo
- Spec seeding (`--spec` flag) from any path
- New-repo scaffolding with branch protection
- Test generation procedure for impl-worker agents

## What Was Built

| Area | Detail |
|------|--------|
| Pipeline stages | spec, plan-milestones, research-worker, design-worker, plan-issues, impl-worker — all operational |
| `composer run` | End-to-end pipeline execution against a target repo |
| `--spec` flag | Seed a target repo with a spec file from any path |
| Target-repo argument | Point composer at any existing or new repo |
| `_scaffold_new_repo` | Scaffold a new GitHub repo with branch protection |
| Test generation | Procedure for impl-worker agents to write tests |
| Spec template | `docs/specs/TEMPLATE.md` + two demo specs (calculator, minesweeper) |
| Glob/Grep allowlist | Added to impl-agent tool allowlist |
| CLAUDECODE guard | Fixed nesting guard, error surfacing, idempotent scaffold |
| Branch protection | Require PRs, block force-push on mainline |

## Non-Goals

Security hardening, multi-model backends, CI deployment, rolling dispatch, crash recovery.
These are deferred to v0.2.x-long-reach.
